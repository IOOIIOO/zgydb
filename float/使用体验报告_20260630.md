## 二、发现的问题

### 🔴 P0 — 严重问题

#### 1. MBTI 四维度分数全部为 0

- **现象**：全选A（偏向E/S/T/J），类型正确判定为ESTJ，但 ei_score / sn_score / tf_score / jp_score 全部返回 0。
- **预期**：全选A时，E/S/T/J 四个维度应有正分（如 E=+10, S=+8 等）。
- **影响**：报告中雷达图无法正确展示维度偏向强度，intensity_level 也受影响（当前为1）。
- **可能原因**：计分函数中 answer 值的大小写匹配问题（JSON 中的 answer 为 `"A"`，计分逻辑可能期望 `"a"`），导致 40 题全部无法进入分支计分。

#### 2. 岗位推荐理由完全模板化（违反任务书核心要求）

- **现象**：10 个岗位的 recommendation_reason 字段内容几乎完全相同：
  > "根据你的能力画像分析，你的 Python 编程能力和数据分析基础与此岗位的核心要求高度匹配。你在校园项目中积累的后端开发经验表明你具备独立完成功能模块的能力，而数学建模竞赛的经历则体现了你的逻辑思维和团队协作能力。建议入职后重点提升企业级开发规范和系统架构设计能力..."

- **问题**：10 个岗位来自不同城市（南京/北京/成都/日本/深圳等）、不同薪资、不同行业，但推荐理由一字不变。
- **违反**：任务书明确要求"推荐理由必须个性化，禁用套话"。
- **可能原因**：`write_recommendation` 函数未传入岗位具体信息，或 prompt 中没有要求针对具体岗位差异化生成。

#### 3. 趋势分析数据来源为假 URL

- **现象**：`info_sources` 返回 `{"url": "https://example.com/report", "title": "行业研究报告"}`。
- **违反**：任务书要求"联网搜索实时获取行业数据"。example.com 是占位假链接。
- **可能原因**：百炼 API 的 `enable_search` 未生效，或 search_trends 函数返回格式中被占位值填充。

#### 4. 发展路径学习资源均为假数据

- **现象**：
  - ISBN: `978-7-115-00000-0`（假的校验位，真实 ISBN 不存在）
  - URL: `https://example.com/docs`（假链接）
  - B站链接: `https://www.bilibili.com/video/example`（假链接）
- **违反**：任务书要求"联网搜索真实学习资源（课程/书籍/GitHub）"。
- **可能原因**：`search_resources` 函数联网搜索失败时回退到占位模板，未做真实性校验。

### 🟡 P1 — 中等问题

#### 5. 能力评估学历提取错误

- **现象**：输入"985计算机专业硕士"，提取结果 education="本科"。
- **预期**：应为"硕士"。
- **可能原因**：qwen3.7-plus 多模态/文本提取 prompt 中缺乏学历字段的精确约束，模型默认偏向"本科"。

#### 6. 岗位推荐 match_score 区分度极低

- **现象**：10 个岗位的 match_score 在 93.2 - 95.0 之间，最大差仅 1.8%。
- **问题**：用户无法根据分数区分岗位匹配程度，失去了推荐排序的意义。
- **可能原因**：Embedding 余弦相似度归一化后分值区间过窄，或 match_score 未用硬维度实际差异加权。

#### 7. 岗位列表 match_score 与详情 overall_match_score 不一致

- **现象**：列表页岗位2247 match_score=95.0，但该岗位详情 overall_match_score=85.0。
- **差异**：相差 10%，两个分数来自不同计算路径，用户困惑。
- **可能原因**：列表页和详情页调用了不同的评分函数（列表用 `recommend` 的整体分数，详情用 `analyze_position_match` 的精确分数）。

### 🟢 P2 — 轻微问题

#### 8. API 返回格式不统一

- 方向列表 `/api/v1/overview/directions` 返回 `{code, data: {items, total}}`（ApiResponse 包裹）。
- 岗位推荐 `/api/v1/recommendation/recommend` 直接返回数组，无 data 包裹。
- 前端需对两种格式分别处理，容易出错。

#### 9. 报告 PDF 未实际生成文件

- 报告 `/api/v1/report/generate` 返回 report_data 入库，但 file_path 字段为空。
- 虽然任务书提到 PDF 导出使用 reportlab/WeasyPrint，但当前未落地文件。
