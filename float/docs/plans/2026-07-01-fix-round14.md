**根因:** `similarity * 100` 在 bge 归一化后天然集中在 0.5~0.7，映射后 49-52，差仅 2 分。

- [ ] **Step 1: 添加非线性拉伸函数并修复 user_emb 重复计算**

```python
def _stretch_score(similarity: float) -> float:
    """将余弦相似度非线性拉伸到 20-100 区间，增大区分度。
    
    bge 归一化后相似度集中在 0.45-0.75，直接 ×100 区分度仅 ±2 分。
    拉伸后：0.3→20, 0.5→50, 0.6→70, 0.7→90, 0.8+→100
    """
    if similarity <= 0.0:
        return 0.0
    elif similarity < 0.3:
        return round(20 + (similarity / 0.3) * 30, 1)
    elif similarity < 0.5:
        return round(50 + ((similarity - 0.3) / 0.2) * 20, 1)
    elif similarity < 0.7:
        return round(70 + ((similarity - 0.5) / 0.2) * 20, 1)
    else:
        return round(min(100.0, 90 + ((similarity - 0.7) / 0.3) * 10), 1)
```

- [ ] **Step 2: 修复 user_emb 重复计算（提到循环外）并应用拉伸**

```python

from app.models.progress import UserProgress

def _check_previous_step(session: Session, user_id: int, step: int) -> None:
    """校验上一步是否已完成，未完成抛出 400"""
    progress = session.exec(
        select(UserProgress).where(UserProgress.user_id == user_id)
    ).first()
    if step == 3:  # Step3 需要 Step2 完成
        if not progress or not progress.step2_completed:
            raise HTTPException(status_code=400, detail="请先完成能力评估（第二步）")
    elif step == 4:  # Step4 需要 Step3 完成
        if not progress or not progress.step3_completed:
            raise HTTPException(status_code=400, detail="请先完成岗位推荐（第三步）")
    elif step == 5:  # Step5 需要 Step4 完成
        if not progress or not progress.step4_completed:
            raise HTTPException(status_code=400, detail="请先完成趋势与发展（第四步）")
```
