# 基于多模态大模型的大学生专业能力分析与职业规划系统设计与实现——AI Agent 开发指令文档


## 文档说明

本文档用于指导 AI Agent 从零开始搭建本系统的完整项目框架。

**核心理念**：先建立完整可运行的系统骨架，所有 AI 能力用假数据占位，确保前端可点、后端可跑、数据库可存、流程可走通。后续确定具体模型后，只需替换假数据模块即可接入真实 AI 能力。

**文档性质**：本文档仅描述"做什么"，不描述"怎么做"。不包含任何具体字段定义、函数名、参数结构或代码示例。


## 一、系统概述

### 1.1 核心定位

基于多模态大模型的大学生专业能力分析与职业规划系统。用户通过完成 MBTI 测评、上传简历或描述自身经历，系统给出能力评估、岗位推荐、行业趋势分析和发展路径规划，最终生成综合报告。

### 1.2 用户操作流程

**系统包含 5 步核心流程 + 1 个独立功能。**

**独立功能（不纳入步骤流程）——快速了解**：

用户在主页可直接点击"快速了解"入口，进入方向浏览页面，查看系统支持的所有职业发展方向。每个方向均可点击查看详细介绍，包含：该方向包含哪些具体岗位、该方向需要什么样的能力与技术、该方向的未来发展趋势如何。此功能无需用户完成任何测评或登录即可浏览。

**5步核心流程（按顺序完成）：**

**第一步：性格分析**
用户完成 MBTI 测评问卷 → 系统计算四维度得分 → 判定人格类型及强度档次 → 匹配对应描述模板 → 返回性格画像、长短处、发展方向倾向。

**第二步：能力评估**
用户上传简历文件或输入对话描述 → 系统提取结构化信息 → 证书竞赛分级 → 输出 3硬维度评分 + 4软维度标签 + 雷达图数据 + 优势短板清单。

**第三步：方向选择**
系统展示可选发展方向列表 → 用户从中选择一个方向 → 系统记录用户选择。

**第四步：深度规划**
系统根据用户选定方向返回：行业趋势分析、岗位推荐列表、短中长期发展路径及学习资源。用户如不满意可返回第三步重新选择。

**第五步：报告生成**
系统汇总前四步全部数据 → 生成完整报告预览 → 用户确认后导出 PDF 文件。

**流程约束**：
- 严格按顺序完成，未完成上一步不能进入下一步
- 第四步支持回退到第三步重新选择方向，重新生成不覆盖旧版本
- 第五步每次重新生成独立保存版本


## 二、系统架构

### 2.1 整体架构

**前端层**：React + TypeScript + Tailwind + ECharts

页面结构：首页（含快速了解入口）→ 主功能页（含步骤进度条）→ 5步流程页面 → 报告预览与导出页

**后端层**：FastAPI + SQLModel

模块划分：快速了解、性格分析、能力评估、岗位推荐、行业趋势、发展路径、报告生成

内部组件：业务逻辑层 + 规则引擎 + 模型接口抽象层

**数据与模型层**：

- MySQL（业务数据存储）
- 模型接口抽象层（所有 AI 能力先做假数据占位）
- 向量检索（待定，预留接口）

### 2.2 关键架构决策

- 所有模型调用通过统一的中间层，业务代码不直接依赖任何模型
- 模型接口先全部返回假数据，确保前后端可在无真实模型时联调
- 规则引擎处理所有确定性逻辑，不调用任何模型


## 三、功能模块详细说明

### 3.1 快速了解模块（独立功能，不纳入步骤流程）

**功能**：

- 在主页提供独立入口
- 展示系统支持的全部职业发展方向
- 每个方向点击后展示详细内容

**每个方向需展示的内容**：
- 该方向包含哪些具体岗位
- 该方向需要什么样的能力与技术
- 该方向的未来发展趋势

**数据来源**：静态预定义数据（与发展方向列表为同一数据源）

**实现方式**：纯数据展示 + 前端路由，不调用任何模型

**边界**：无需登录即可浏览，不依赖任何用户数据，不产生任何记录

### 3.2 性格分析模块

**功能**：

- 提供固化的 MBTI 四维度测评题目
- 用户逐题作答后，系统计算各维度得分
- 根据得分判定 16 种人格类型之一及强度档次
- 从 48 套描述模板中匹配对应画像

**输出内容**：人格类型标签、四维度得分、长短处列表、综合画像描述、发展方向倾向列表

**实现方式**：纯规则引擎 + 查表，不调用任何模型

**边界条件**：测评结果标注"仅供参考"，不诊断心理疾病，性格结果不参与能力评分计算

### 3.3 能力评估模块

**功能**：

- 接收用户上传的简历文件或对话文本
- 提取结构化信息：学历、课程知识、技能工具、证书、竞赛、项目经历、社团经历
- 对证书进行分级，对竞赛进行分级
- 输出 3硬维度评分和 4软维度标签

**3硬维度说明**（参与岗位匹配计算）：

- 知识：专业课程、理论、行业术语的掌握程度
- 工具：具体软件、编程语言、硬件仪器的熟练度
- 项目：项目复杂度、完整度、个人角色与贡献

**4软维度说明**（不参与匹配公式，仅外围控制）：

- 逻辑与解决问题：根据项目描述中的动作词和量化成果推断
- 沟通与协作：根据团队角色和跨角色经历推断
- 证书/竞赛含金量：由分类模型确定性分级
- 学习能力：根据跨界技能和自学经历推断，作为后置微调系数

**模型调用需求**（全部先做假数据占位）：

- 多模态大模型：直接从 PDF/图片提取结构化信息
- 通用大模型：从对话文本提取结构化信息
- 分类模型：证书分级 + 竞赛分级
- 通用大模型：3硬维度评分 + 4软维度标签推断

**边界条件**：不存储简历原文，不代替 HR 做录用判断

### 3.4 岗位推荐模块

**功能**：

- 基于用户 3硬能力画像与岗位库进行匹配
- 学历硬门槛过滤
- 性格标签做极端情况后置过滤
- 输出推荐岗位及个性化推荐理由

**推荐逻辑顺序**：

1. 学历硬门槛过滤
2. 文本向量化 + 余弦相似度排序
3. 性格约束过滤（查询岗位的性格排斥配置）
4. 大模型撰写个性化推荐理由

**性格约束逻辑**：性格不参与向量计算，仅做后置规则过滤。查询岗位配置中是否排斥用户的人格类型，或是否触发极端特质阈值。

**模型调用需求**（全部先做假数据占位）：

- Embedding 模型：文本向量化 + 余弦相似度
- 通用大模型：撰写推荐理由

**边界条件**：岗位推荐与行业趋势分析解耦，互不依赖

### 3.5 行业趋势与发展路径模块

**功能**：

- 针对用户选定岗位搜索行业趋势数据
- 搜索真实学习资源
- 生成短中长期发展路径

**趋势分析必须包含的维度**：

- 3-5 年发展趋势
- 技术变革对岗位的具体影响
- 地域需求差异
- 薪资走向变化
- 入门门槛变化趋势
- 落到用户个体层面：技能缺口、3 年后竞争力、转型风险

**发展路径结构**：

- 短期（6个月-1年）：入门技能 + 具体学习资源
- 中期（1-3年）：中级成长路径与技能提升
- 长期（3-5年）：职业进阶方向与高级能力要求

**模型调用需求**（全部先做假数据占位）：

- 通用大模型 + 联网搜索：行业趋势搜索
- 通用大模型 + 联网搜索：学习资源搜索
- 通用大模型：发展路径个性化填充

**边界条件**：资源推荐必须提供 URL 或 ISBN 证据，不推荐盗版或已下架资源

### 3.6 报告生成模块

**功能**：

- 汇总前四步全部数据
- 模板引擎组装结构化文档
- 大模型润色语言
- 导出 PDF

**报告结构**：封面 → 性格画像 → 能力雷达图与评分 → 长处短板 → 推荐岗位及理由 → 发展路径 → 行业趋势分析 → 信息来源附录

**模型调用需求**（全部先做假数据占位）：

- 通用大模型：报告语言润色

**边界条件**：报告首页标注"仅供参考"，每次重新生成独立保存版本号


## 四、数据库设计原则

AI Agent 自行设计表结构，满足以下数据存储需求。

**数据库设计必须遵守以下原则**：

1. **每个表必须有明确的业务用途**：对照本文档第三部分各模块的功能描述，每个表必须对应至少一个功能模块的数据存储需求
2. **每个字段必须有明确的调用场景**：设计字段时，必须明确该字段在哪个接口的哪个环节被读写。如果一个字段在业务流程中从未被使用，则为无效字段，不应存在
3. **不允许出现"备用字段"或"预留字段"**：不要设计当前用不上、未来"可能"会用到的字段
4. **表之间关系必须清晰**：外键关联必须有明确的业务含义

**各模块数据存储需求**：

| 表用途 | 对应模块 | 需要存储的内容 |
|:---|:---|:---|
| 用户认证 | 通用 | 用户名、邮箱、密码哈希 |
| 性格测评结果 | 性格分析 | 用户、人格类型、八维度得分、强度档、长短处、描述、方向倾向 |
| 能力画像 | 能力评估 | 用户、学历、3硬评分、4软标签、评分依据、优势短板 |
| 证书记录 | 能力评估 | 用户、证书名、档次、档次名称、得分 |
| 竞赛记录 | 能力评估 | 用户、竞赛名、级别、级别名称、加分 |
| 岗位数据 | 岗位推荐 | 岗位标题、描述、方向、学历要求、薪资范围、性格约束配置 |
| 岗位推荐记录 | 岗位推荐 | 用户、岗位、匹配度、推荐理由 |
| 趋势分析 | 行业趋势 | 用户、岗位方向、完整趋势数据、风险提示、信息来源 |
| 发展路径 | 发展路径 | 用户、岗位、短中长期路径、资源列表、版本号 |
| 综合报告 | 报告生成 | 用户、完整报告数据、PDF路径、版本号 |
| 方向数据 | 快速了解 + 方向选择 | 方向名称、岗位举例、所需技术、发展趋势描述、性格约束配置 |

**注意**：方向数据表同时服务于"快速了解"功能和第三步"方向选择"，两者使用同一数据源。


## 五、静态数据准备

### 5.1 静态数据列表

AI Agent 需要准备以下数据，以 JSON 或 SQL 初始化数据形式提供：

**数据一：48套 MBTI 描述模板**
16种人格类型 × 3档强度，每套含类型标签、长处列表、短板列表、综合画像描述、发展方向倾向。

**数据二：发展方向完整数据（核心静态数据）**
此数据同时服务于"快速了解"功能和第三步的"方向选择"，两者使用同一数据源。

每个方向包含：
- 方向名称
- 具体岗位举例列表
- 所需能力与技术标签
- 未来发展趋势简述
- 性格约束配置（排除人格列表、排除特质列表）

数量约 30 个方向，覆盖主流就业方向。

### 5.2 重要说明

发展方向数据是**静态预定义数据**，一次性建立，持续维护。

- "快速了解"功能：直接展示这些数据，供用户浏览查阅
- 第三步"方向选择"：用户从同一份方向列表中选择一个
- 第四步的详细分析：基于用户选择的方向，由大模型实时生成


## 六、模型接口预留规范

### 6.1 需要预留的模型接口（共5个模型，11个功能点）

| 编号 | 功能描述 | 调用位置 | 假数据需返回的内容 |
|:---|:---|:---|:---|
| M1 | 多模态大模型：直接提取 PDF/图片中的结构化信息 | 能力评估-上传简历 | 模拟的完整简历结构化数据 |
| M2-1 | 通用大模型：从对话文本提取结构化信息 | 能力评估-对话描述 | 模拟的结构化提取结果 |
| M2-2 | 通用大模型：知识/工具/项目三维度评分 | 能力评估-评分环节 | 三个分数及评分依据 |
| M2-3 | 通用大模型：逻辑/沟通/学习标签推断 | 能力评估-标签环节 | 三个标签及推断依据 |
| M2-4 | 通用大模型：撰写岗位推荐理由 | 岗位推荐-写理由 | 完整的推荐理由文本 |
| M2-5 | 通用大模型：报告语言润色 | 报告生成-润色环节 | 润色后的报告文本 |
| M3 | 通用大模型+联网搜索：搜索趋势或资源 | 趋势分析 + 路径生成 | 模拟趋势数据或资源列表 |
| M4 | 分类模型：证书分级 + 竞赛分级 | 能力评估-分级环节 | 证书→档次；竞赛→级别 |
| M5 | Embedding模型：文本向量化 + 余弦相似度 | 岗位推荐-匹配环节 | 模拟相似度排序结果 |

### 6.2 假数据管理三条铁律

**铁律一**：所有返回假数据的函数，函数名必须以 FAKE_ 开头

**铁律二**：所有假数据函数统一存放在 backend/app/services/fake_data/ 目录下

**铁律三**：每个假数据函数必须包含注释，说明功能描述、TODO（替换为真实模型）、预期模型类型、调用位置

### 6.3 假数据目录结构

```
backend/app/services/fake_data/
├── __init__.py
├── README.md（说明此目录为假数据，正式部署时删除）
├── multimodal.py
├── general_llm.py
├── classifier.py
├── embedding.py
└── search.py
```

### 6.4 模型中间层要求

所有模型调用通过一个统一中间层：

- 开发阶段：从 fake_data 导入所有 FAKE_ 函数
- 正式阶段：替换为真实模型导入
- 业务代码永远只调用中间层，不直接依赖 fake_data 或真实模型


## 七、规则引擎功能

以下规则由纯代码实现，不调用任何模型：

1. 学历硬门槛过滤
2. 证书分数越档校验

4. 性格约束过滤（查询岗位性格排斥配置，剔除不匹配岗位）

1. 所有接口先返回假数据确保流程跑通

4. 所有假数据在 fake_data 目录
5. 模型调用通过中间层统一管理
6. 不硬编码任何具体模型名称
7. 数据库配置在 .env，提供 .env.example
8. 前端 API 地址环境变量配置
9. 完整类型注解
